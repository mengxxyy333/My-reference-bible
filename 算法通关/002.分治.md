# 归并分治

***前置知识：归并排序、递归***

​	归并分治的想法也很妙，但是能解决的问题由简到难，很多很多。这个和递归差不多，都需要去好好体会。

## 原理

​	1、思考一个问题大范围的答案，是否等于，左半部分的答案 + 右半部分的答案 + 跨越左右产生的答案。

​	2、计算“跨越左右产生的答案”时，如果加上左右各自有序这个设定，会不会获得计算的便利性。

​	3、如果以上两点都成立，那么该问题很可能被归并分治解决。

​	4、求解过程需要加入归并排序，因为要让左右有序来获得计算的便利性。

## 两道简单题目

### 小和问题

​	题目链接：https://www.nowcoder.com/practice/edfe05a1d45c4ea89101d936cac32469

​	左半边的小和 + 右半边的小和 + 跨越左右产生的小和就是答案，符合我们上面说的第一条，同时，左右部分各有序，也能帮助加速我们的计算过程，符合上面第二条，所以可以用归并分治解决。

​	AC代码如下：

```c++
#include <bits/stdc++.h>
using namespace std;

#define MAXN 100005
typedef long long ll;

int n;
int arr[MAXN], help[MAXN];

// 经典归并排序的merge过程，加上了统计跨越左右产生的答案而已！
ll merge(int l, int m, int r) {
    ll ans = 0, sum = 0;
    for(int j = m + 1, i = l; j <= r; j++) {
        while(i <= m && arr[i] <= arr[j]) {
            sum += arr[i++];
        }
        ans += sum;
    }

    int i = l;
    int a = l, b = m + 1;
    while(a <= m && b <= r) {
        help[i++] = arr[a] <= arr[b] ? arr[a++] : arr[b++];
    }
    while(a <= m) {
        help[i++] = arr[a++];
    }
    while(b <= r) {
        help[i++] = arr[b++];
    }
    for(int j = l; j <= r; j++) arr[j] = help[j];

    return ans;
}

// 求区间[l, r]上的小和
ll smallsum(int l, int r) {
    if(l == r) return 0;
    int m = (l + r) >> 1;
    // 左半部分的小和、右半部分的小和、跨越左右的小和
    return smallsum(l, m) + smallsum(m + 1, r) + merge(l, m, r);
}

int main() {
    cin >> n;
    for(int i = 0; i < n; i++) cin >> arr[i];

    cout << smallsum(0, n - 1);

    return 0;
}
```

### 翻转对问题

​	题目链接：https://leetcode.cn/problems/reverse-pairs/

​	分析同上，代码也几乎一样！力扣AC代码如下：

```c++
class Solution {
public:
    int help[100005];

    int merge(vector<int>& nums, int l, int m, int r) {
        int ans = 0;
        for(int i = l, j = m + 1; i <= m; i++) {
            while(j <= r && (long long) nums[i] > (long long) 2 * nums[j]) {
                j++;
            }
            ans += j - m - 1;
        }

        int i = l;
        int a = l, b = m + 1;
        while(a <= m && b <= r) {
            help[i++] = nums[a] <= nums[b] ? nums[a++] : nums[b++];
        }
        while(a <= m) {
            help[i++] = nums[a++];
        }
        while(b <= r) {
            help[i++] = nums[b++];
        }
        for(int i = l; i <= r; i++) {
            nums[i] = help[i];
        }
        return ans;
    }

    int f(vector<int>& nums, int l, int r) {
        if(l == r) return 0;
        int m = (l + r) >> 1;
        return f(nums, l, m) + f(nums, m + 1, r) + merge(nums, l, m, r);
    }

    int reversePairs(vector<int>& nums) {
        return f(nums, 0, nums.size() - 1);
    }
};
```

​	上述两个问题的时间复杂度都为O( n * logn )，因为就是归并排序的过程，加了一步找答案的步骤，而该步骤与merge过程一样都串一遍[l, r]单位上的数字而已，增加的只是常数时间，并不增加复杂度。

## 补充

​	一些用归并分治解决的问题，往往可以用线段树、树状数组解决，时间复杂度也是最优的。本节两个题目算是简单题，归并分治还能解决一些难题，比如平面空间最近点对问题，该问题会在后面介绍，感兴趣读者也可自行阅读，来自《算法导论》。
