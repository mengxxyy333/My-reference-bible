# 异或运算的骚操作

***前置知识：二进制与位运算***

## 开篇--一个有趣的问题

​	袋子中一共有a个白球，b个黑球，每次从袋子中拿两个球，每个球每次被拿出的机会均等。如果拿出的是两个白球或者两个黑球，那么往袋子中重新放入一个白球，如果拿出一黑一白，放入一个黑球。这样相当于每次拿2放1，最终袋子里的球一定会剩下一个。请问最终袋子中剩余黑球的概率是多少？

​	答案：如果黑球数量为偶数，最终剩余黑球的概率是0；如果黑球数量为奇数，最终剩余黑球的概率是1。

​	想要解决这个问题，就需要了解异或运算，那就先来看看异或运算有什么样的性质。

## 异或运算性质

***注：所有结论都可以由结论1推出，但是做题使用最多的是结论4。***

​	1、异或运算就是无进位相加；

​	2、异或运算满足交换律、结合律；

​	3、0 ^ n = n，n ^ n = 0；

​	4、整体异或结果如果为x，该整体中某个部分的异或结果为y，那么剩余部分的异或结果为x ^ y，相当于y的那部分再次异或出去就是0了，而0异或任何数结果就是该数本身。简单的结论就是：a ^ b ^ b = a，相当于把b给“挑出去”，异或出去就是0了。

​	5、如果了解逻辑运算（例如数电课程中），与异或概念截然相反的还有一个同或，同或就是：相同为1，不同为0，正好与异或相反。当然C++中并没有对应的同或运算符，我们可以通过异或得到a和b同或的结果：~(a ^ b) 或 (a & b) | (~a & ~b)，a异或b再取反即为a和b同或，但是取反操作会连带符号位，影响最终结果，需要注意这一点。

## 开篇问题答案

​	至此，开篇提到的问题就可以解释了，我们把白球看作数字0，黑球看作数字1，取两个白球或两个黑球放入一个白球，是不是就相当于0 ^ 0 = 1 ^ 1 = 0，一黑一白就相当于1 ^ 0 = 1，整个过程就相当于：所有球转换为对应数字后，总的异或结果。那么就只跟1的奇偶性有关了。

## 异或运算的更多骚操作

### 交换两个数

​	不用swap函数，用异或完成两数的交换：

```c++
int a = 10, b = -2333;
a = a ^ b;
b = a ^ b;
a = a ^ b;
```

​	分析一下为什么。假设a = 甲， b = 乙，第一句执行后，a = 甲 ^ 乙。第二句执行时，b = a ^ b = （甲 ^ 乙） ^ 乙，根据上面的性质，此时b = 甲。最后一句执行后，a = a ^ b = （甲 ^ 乙）^ 甲 = 乙。完成a与b的交换。

​	什么时候适用，什么时候不适用呢？在a和b有独属于自己的空间时，适用，如何a和b的空间是同一个，那么这样做会把a和b都刷成0，所以这种写法看看就好，实际写代码还是swap函数好。

### 比较最大值

​	题目链接：https://www.nowcoder.com/practice/d2707eaf98124f1e8f1d9c18ad487f76

​	不用判断语句和比较语句，返回两个数的最大值，当然和上面一样的，看看就好。

​	先写一个函数，将某个数字的符号位取出：

```c++
// 返回n的符号，负数返回0，非负返回1。有点反直觉，后面看原因。
int sign(int n) {
    return (n >>> 31) ^ 1;
}
// 或者使用带符号右移运算符，这样写：
int sign(int n) {
    return ((n >> 31) & 1) ^ 1;
}
```

​	比较大小可以作差或者作商比较，代码中作差要方便，看下面的代码：

```c++
int getmax1(int a, int b) {
    int c = a - b; // 看c的符号即可
    int returnA = sign(c); // c是负数，说明a<b，returnA=0, returnB=1, 看最后一行，此时返回b，是最大值，反之也成立
    int returnB = returnA ^ 1;
    return a * returnA + b * returnB;
}
```

​	但是这个函数有一个缺点：a - b的结果可能会溢出。什么情况a - b会溢出呢？a为正，b为负，结果可能溢出；而a与b如果同号，是不可能溢出的。我们增加一些条件来判断这个事情就可以了，代码及注释如下：

```c++
int getmax2(int a, int b) {
    int c = a - b; // 此时c仍可能溢出，但是无所谓
    
    // 获取a,b,c的符号，非负返回1，负数返回0.
    int sa = sign(a), sb = sign(b), sc = sign(c);
    
    int diffAB = sa ^ sb; // 判断a与b的符号是否不同，不一样为1，一样为0
    int sameAB = diffAB ^ 1; // 判断a与b的符号是否相同，一样为1，不一样为0，正好与diffAB相反
    
    // 我们来看什么时候返回a:
    // 1、a非负，b为负数。此时diffAB = 1且sa = 1，sameAB = 0，后面项就忽略了
    // 2、a与b同号且c为正，此时没有溢出风险，diffAB = 0，sameAB = 1，sc = 1
    // 两种情况至少有一个成立，returnA才为1，下面才能返回a
    int returnA = diffAB * sa + sameAB * sc;
    int returnB = returnA ^ 1; // 除了返回a的情况，就是返回b
    
    return a * returnA + b * returnB;
}
```

### 找到缺失的数字

​	题目链接：https://leetcode.cn/problems/missing-number/

​	将[0, n]整体异或起来，再异或出现过的n个数字，相当于将出现过的数字都异或出去，得到的结果就是未出现的数字。力扣AC代码：

```c++
int missingNumber(vector<int>& nums) {
    int eorAll = 0, eorHash = 0;
    for(int i = 0; i < nums.size(); i++) {
        eorAll ^= i;
        eorHash ^= nums[i];
    }
    eorAll ^= nums.size();
    return eorAll ^ eorHash;
}
```

### 找出唯一一个出现奇数次的数字

​	题目链接：https://leetcode.cn/problems/single-number/

​	将所有数字异或起来即可，出现偶数次数的数字都异或为0，假设出现奇数次数的数字出现了n次，那么n-1次，也就是偶数次，也全异或为0，只剩下0^该数字，所以剩下的就是答案，力扣AC代码：

```c++
int singleNumber(vector<int>& nums) {
    int eor = 0;
    for(int &x : nums) eor ^= x;
    return eor;
}
```

### Brian Kernighhan算法

​	该算法用于提取一个数字二进制状态下最右侧的1，也就是最低位的1。例如：n = 01101000，想要提取的状态为：00001000。我们将n的所有位取反，~n = 10010111，再加1，~n + 1 = 10011000，与原来的n按位与即可。这样操作的效果是：比最低位的1高位的那些状态全取反，比最低位1低的所有0变成1，通过加1再变回0，而这个1取反变成0，通过下面的进位变成1，这样二者状态按位与下来，只剩下最低位的1了。而一个正整数n取反再加1，在计算机中不就是这个数字的相反数吗？代码如下：

```c++
int lowbit(int n) {
    return n & (-n);
}
```

​	该函数用于解决下面的问题，同时这个函数在以后的课程中尤其是树状数组中，会大量用到，很重要的一个算法，或者说小技巧。

### 找出唯二出现奇数次的数字

​	题目链接：https://leetcode.cn/problems/single-number-iii/

​	先将所有数字异或起来，得到的结果应该是a ^ b。这时能不能想办法求出a和b其中一个数字呢？可以，我们取出结果中最低位的1（其实任意一个位的1都可以，只不过最低位的1容易写代码），结果是1，说明在这位上，a与b不同，即a与b分属于两个阵营，一个是该位为1，一个是该位为0。我们只需要另外的一个变量来记录其中一个阵营的总的异或结果，就能求出a与b其中的一个，因为该阵营中其他的数字一定都被异或为0，再用a ^ b的结果异或该结果，就能求出另一个了。力扣AC代码：

```c++
vector<int> singleNumber(vector<int>& nums) {
    int eor1 = 0; // 总的异或结果
    for(int &x : nums) eor1 ^= x;

    int eor2 = 0; // 总的该位为0的异或结果，能够求出a或者b，但是具体是谁不知道
    unsigned int rightone = (unsigned int)eor1 & (-(unsigned int)eor1);
    for(int x : nums) {
        if(((unsigned int)x & rightone) == 0) {
            eor2 ^= x;
        }
    }
    return {eor2, eor1 ^ eor2};
}
```

### 找出唯一出现少于m次的数字，其余数字均出现m次

​	题目链接：https://leetcode.cn/problems/single-number-ii/

​	本题用一个长度为32的数组，分别记录所有数字在0到31位上1的出现次数，如果目前所有数字出现次数均为m次，那么它在对应二进制有1的位的贡献就是m，最终统计出来的cnt[i]（即第i位上所有数字二进制状态下1的个数）一定是m的整数倍。一旦出现不是m的整数倍的情况，那就是要找的数字出现了，在该位贡献了不足m个1，所以该位的1一定是要找的数字贡献的，将所有的1或到答案中去，就是答案。力扣AC代码：

```c++
int singleNumber(vector<int>& nums) {
    int cnt[32]{};
    for(int x : nums) {
        for(int i = 0; i < 32; i++) {
            cnt[i] += (x >> i) & 1; // 统计当前数字x第i位是否为1
        }
    }

    int ans = 0, m = 3; // 本题中m为3，答案出现次数为1
    for(int  i = 0; i < 32; i++) {
        if(cnt[i] % m) ans |= (1 << i); // 第i位的1的个数不是m的整数倍，那么一定是答案在该位贡献了1，或到ans中保存
    }
    return ans;
}
```
