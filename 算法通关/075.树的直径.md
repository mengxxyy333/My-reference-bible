# 树的直径

​	***前置知识：链式前向星、树型dp***

## 定义

​	树上距离最远的两个点形成的路径就叫做树的直径。路径总长度叫做树的直径的长度。

## 相关结论

​	前提是树上的边权都为正：

​		1、如果有多条直径，那么这些直径一定拥有公共的中间部分，可能是一个公共点，也可能是一段公共路径；

​		2、树上任意一点，相隔最远的点的集合，直径的两端至少有一个在其中。

## 两种求树的直径的方式

​	1、两次 dfs 的方式，该方式适用于边权非负的树。

​		优点：不仅能得到直径长度，还能得到直径上的所有点等等很多信息

​		缺点：遍历两次树

​		也可以用两次 bfs 来实现，但是 dfs 是更常用的。

​	2、树型dp的方式，适用于所有树。

​		优点：遍历一次树

​		缺点：只能得到直径的长度

### 两次dfs（适用于边权非负的树）

​	题目链接：https://www.luogu.com.cn/problem/P8602

​	过程很简单，第一次 dfs 从树根出发，找到离树根最远的节点 u。第二次从 u 出发，找到离 u 最远的节点 v。这时，从 u 到 v 的路径就是这棵树的直径。证明略。

​	有些题目可能需要用到直径的端点、以及直径路径上的点的信息，所以此处用两个数组：dist[i] 表示当前头节点到节点 i 的距离，last[i] 表示这次遍历过程中，i 节点的父节点是哪个。两次 dfs 均更新二者，但是只有第二次更新，last 数组才能找到整条路径。详细代码见下：

```c++
#include <bits/stdc++.h>
using namespace std;

#define MAXN 100005
#define MAXM 100005
const int mod = 1e9 + 7;
typedef long long ll;

int n;
int s, e, len;
int dist[MAXN], last[MAXN];

int head[MAXN];
int to[MAXN << 1];
int nxt[MAXN << 1];
int weight[MAXN << 1];
int cnt = 1;
void addedge(int u, int v, int w) {
	nxt[cnt] = head[u];
	to[cnt] = v;
	weight[cnt] = w;
	head[u] = cnt++;
}

void dfs(int u, int fa, int w) {
	last[u] = fa;
	dist[u] = dist[fa] + w;
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			dfs(to[e], u, weight[e]);
		}
	}
}

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);
	
	cin >> n;
	for(int i = 1, u, v, w; i <= n - 1; i++) {
		cin >> u >> v >> w;
		addedge(u, v, w);
		addedge(v, u, w);
	}

	dfs(1, 0, 0);
	s = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[s]) {
			s = i;
		}
	}

	dfs(s, 0, 0);
	e = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[e]) {
			e = i;
		}
	}
	
	len = dist[e];
	cout << (21 + len) * len / 2;

	return 0;
}
```

### 树型dp（适用于所有树）

​	题目链接：https://www.luogu.com.cn/problem/U81904

​	需要设置两个数组：dist[u] 表示：必须从 u 开始向下走，能走出的最大距离，可以不选任何边，那 dist[u] 初始值就是0。ans[u] 表示：路径一定要包含 u 的情况下能得到的最大路径长度，ans[u] 的初始值同样为0，看作是自己到自己最优。

​	那么对于任意节点 u，如何更新呢？先更新 ans[u]，再更新 dist[u]。更新 ans[u] 的策略是：dist[u] + dist[子] + u 到子的边权，维护出一个最大值。此时 dist[u] 也同步更新，维持最大值。更抽象的来说，dist[u] 记录的是，之前某一条路到 u 的权值最大值，ans[u] 就是 dist[u] 加上新遍历的另一条支路能获得的权重。整体来看，找到路径必须经过节点 u 时，可能一条向下走的路都没有，即不通过 u 去往 u 的子树最优，或者只走 u 的一条或者两条支路，最多只有两条，因为是直径，经过一个点不可能超过两条分支。而 dist 就用于记录之前的 u 的支路的最大值，ans[u] 每一步通过这个 dist[u] 再去找其他支路，看看组合到一起能否有更大的值。

​	上面这些只是针对与某一个点，如果对于整棵树的节点自底向上都更新好 dist 和 ans 的值，那么最后的最大值就是树的直径的长度，因为树的直径必经过某个点，而我们已经求出了每个点作为最高点（转折点或者头节点），最长的路径长度，整个过程维护一个最大值就行了。详细AC代码如下：

```c++
#include <bits/stdc++.h>
using namespace std;

#define MAXN 500005
#define MAXM 100005
const int mod = 1e9 + 7;
typedef long long ll;

int n;
int len = INT_MIN;
int dist[MAXN], ans[MAXN];

int head[MAXN];
int to[MAXN << 1];
int nxt[MAXN << 1];
int weight[MAXN << 1];
int cnt = 1;
void addedge(int u, int v, int w) {
	nxt[cnt] = head[u];
	to[cnt] = v;
	weight[cnt] = w;
	head[u] = cnt++;
}

void dfs(int u, int fa) {
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			dfs(to[e], u);
		}
	}
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			ans[u] = max(ans[u], dist[u] + dist[to[e]] + weight[e]);
			dist[u] = max(dist[u], dist[to[e]] + weight[e]);
		}
	}
}

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);
	
	cin >> n;
	for(int i = 1, u, v, w; i <= n - 1; i++) {
		cin >> u >> v >> w;
		addedge(u, v, w);
		addedge(v, u, w);
	}

	dfs(1, 0);
	for(int i = 1; i <= n; i++) {
		len = max(len, ans[i]);
	}

	cout << len;

	return 0;
}
```

​	这部分代码回头看，ans 数组是没必要的，因为就算收集了每个节点的 ans，最后也是PK出最大值，莫不如在更新 ans 数组时，就更新最大直径长度了，同时 dist 数组的更新也不依赖于 ans 数组，这样可以省掉一个数组，ans 只是为了让大家理解转移逻辑。优化代码如下：

### 所有直径的公共部分有几条边

​	题目链接：https://www.luogu.com.cn/problem/P3304

​	这道题目需要明确一个道理：假设有一条已经确定端点的直径 s --> e，那么路径上的所有点去dfs求走过的最大距离，不能走直径上的点，这样得到的最大距离一定小于等于这个点到 e 的距离，因为如果大于，那么这条直径找的就是错误的。小于是正确的，再来分析等于的情况，也就是除了找到的这条直径，从这个点走向其他子树仍然可以得到直径长度，这就意味着新发现的这个路径也是直径的一部分，从当前点分开的不同直径，同样也说明：当前这个点就是两条直径公共部分的右端点。那么多条直径呢？公共部分的右端点，就是符合条件的点的最左侧的那个点，符合什么条件？就是这个点到 e 的距离，等于其以该点为头，遍历非直径上的点走出的最大距离。公共部分的左端点呢？其实就是其他子树走出的最大距离等于该点到 s 的距离，满足这个条件的最右端点。详细代码如下：

```c++
#include <bits/stdc++.h>
using namespace std;

#define MAXN 200005
#define MAXM 100005
const int mod = 1e9 + 7;
typedef long long ll;

int n;

int head[MAXN];
int nxt[MAXN << 1];
int to[MAXN << 1];
int weight[MAXN << 1];
int cnt = 1;
void addedge(int u, int v, int w) {
	nxt[cnt] = head[u];
	to[cnt] = v;
	weight[cnt] = w;
	head[u] = cnt++;
}

int s, e;
ll dist[MAXN], last[MAXN];
ll diameter;
bool daimeterPath[MAXN];
int commonEdges;

void dfs(int u, int fa, ll w) {
	last[u] = fa;
	dist[u] = dist[fa] + w;
	for(int e = head[u]; e != 0; e = nxt[e]){
		if(to[e] != fa) {
			dfs(to[e], u, weight[e]);
		}
	}
}

// 求每个点的子树能走到的最大距离，但是不能走路径上的点
// c就是目前为止走过以u为头的树的最大权值
ll maxDistExceptDiameter(int u, int fa, ll c) {
	ll ans = c;
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(!daimeterPath[to[e]] && to[e] != fa) {
			ans = max(ans, maxDistExceptDiameter(to[e], u, c + weight[e]));
		}
	}
	return ans;
}

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);

	cin >> n;
	for(int i = 1; i <= n - 1; i++) {
		int u, v, w; cin >> u >> v >> w;
		addedge(u, v, w);
		addedge(v, u, w);
	}

	// 先两次dfs找到一条直径的端点以及长度
	dfs(1, 0, 0);
	s = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[s]) {
			s = i;
		}
	}
	dfs(s, 0, 0);
	e = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[e]) {
			e = i;
		}
	}
	diameter = dist[e];

	// 倒着找出直径上的所有点，标记，后续节点不能再走路径上的点
	for(int i = e; i != 0; i = last[i]) {
		daimeterPath[i] = true;
	}

	// 对于每个路径上的点，去其他子树(除了路径上的点)计算最大距离
	// 如果这个最大距离等于这个点到e的距离，那么这个点一定是另外一条直径的右端点
	// 而对于所有直径的公共右端点，需要找到最左侧的符合的条件的点
	// 如果这个最大距离等于这个点到s的距离，那么这个点一定是另外一条直径的左端点
	// 而对于所有直径的公共左端点，需要找到最右侧的符合的条件的点
	// 既然是从e倒着推到s，那么左端点就是最先遇到的符合条件的点
	int l = s, r = e;
	for(int i = last[e]; i != s; i = last[i]) {
		ll maxdist = maxDistExceptDiameter(i, 0, 0);
		if(maxdist == diameter - dist[i]) {
			r = i;
		}
		if(maxdist == dist[i] && l == s) {
			l = i;
		}
	}

	// l == r 说明只有一点公有，公共边自然为0
	if(l != r) {
		commonEdges = 1;
		for(int i = last[r]; i != l; i = last[i]) {
			commonEdges++;
		}
	}

	cout << diameter << "\n" << commonEdges;

	return 0;
}
```

### 合并两棵树后直径尽可能短

​	题目链接：https://www.luogu.com.cn/problem/P2195

​	这道题目首先肯定需要并查集，因为题目给出的是一个森林，而不是完整的树；其次在操作2时还有合并的操作。对于每棵树，我们先计算内部的直径长度，然后保存到 diameter 数组中，加入这棵树的代表节点是 u，那就存到 diameter[u] 中，之后关于这棵树的所有节点的查询，都可以找到 u，然后输出 diameter[u]，这就完成了操作一的查询。对于操作二，合并即两棵树的合并，那么只需要两棵树的代表节点合并，然后再将新的代表节点的 diameter 更新好就可以，那么合并后两棵树的直径如何最短？这里有一个贪心的点：那就是连接时选择两个直径的中间节点连接，假设原始直径长度为 len1 和 len2，那么合并后的直径就可能是：向上取整(len1 / 2) + 向上取整(len2 / 2) + 1，这里向上取整是因为直径是取最大值，假如 len2 长度是5，那么连接后，len2这边会被计入直径长度的一定是3这边而不是2那边，否则不符合直径定义；后面+1是因为新连接了一条边。为什么说新直径可能是这个值？因为直径一定是最长的，但是连接之后，有可能最大的路径仍然保持在 len1 或者 len2，故合并后的直径长度要在三者中取最大值。其他细节看AC代码：

```c++
#include <bits/stdc++.h>
using namespace std;

#define MAXN 300005
#define MAXM 100005
const int mod = 1e9 + 7;
typedef long long ll;

int n, m, q;
int dist[MAXN];
int diameter[MAXN];

int head[MAXN];
int nxt[MAXN << 1];
int to[MAXN << 1];
int cnt = 1;
void addedge(int u, int v) {
	nxt[cnt] = head[u];
	to[cnt] = v;
	head[u] = cnt++;
}

int father[MAXN];
int find(int x) {
	if(x != father[x]) {
		father[x] = find(father[x]);
	}
	return father[x];
}

void dfs(int u, int fa) {
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			dfs(to[e], u);
		}
	}
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			// 直径有可能不经过u，所以需要与其子节点发现的最大距离作比较
			diameter[u] = max(diameter[u], max(diameter[to[e]], dist[u] + dist[to[e]] + 1));
			dist[u] = max(dist[u], dist[to[e]] + 1);
		}
	}
}

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);
	
	cin >> n >> m >> q;
	for(int i = 1; i <= n; i++) {
		father[i] = i;
	}
	for(int i = 1; i <= m; i++) {
		int u, v; cin >> u >> v;
		addedge(u, v);
		addedge(v, u);
		u = find(u);
		v = find(v);
		father[u] = v;
	}
    
    // 对于每棵树，计算其直径长度
	for(int i = 1; i <= n; i++) {
		if(i == father[i]) {
			dfs(i, 0);
		}
	}

	for(int i = 1; i <= q; i++) {
		int op, x, y;
		cin >> op;
		if(op == 1) {
			cin >> x;
			x = find(x);
			cout << diameter[x] << "\n";
		}
		else {
			cin >> x >> y;
			x = find(x);
			y = find(y);
			if(x != y) {
				father[x] = y;
				diameter[y] = max((diameter[x] + 1) / 2 + (diameter[y] + 1) / 2 + 1, max(diameter[x], diameter[y]));
			}
		}
	}

	return 0;
}
```

### 新建边使得走过的路最短

​	题目链接：https://www.luogu.com.cn/problem/P3629

​	这道题目较难了，需要一些分析。如果不连边，那么走过的边数一定是2倍边数，每条边都需要走两遍。正是因为连接了边，所以才会节省路径，那么一定是形成了环，且环最大，省下的边数（即只走一次的边数）是最多的，这个读者可以尝试画一棵树，连接一个环走一走，环上的边一定可以只走一次。很自然的想到树的直径，在两个端点处连接，可以组成最大的环。那么这时走过的边数就是：不加边时走过的边数 - 直径上的边数 + 1，加这个1是因为新建了一条边需要通过。这是 k = 1的情况，但是题目给出的 k 也可能为2，如果连两条边，如何最大的节省走过的边呢？很自然想到：找到最大的环和次大的环，这样最省。大体思路是正确，但是有几处小细节：1、两个环的位置。这两个环可能有边重合，也可能只有一个点重合，读者可以分别画出一棵树，对于只有一个点重合的，其实每条边可以只走一遍，这是最省的情况；那么有边重合的时候呢？读者可以验证：两个环重合的边必走两次，所以重合的边越少越好。这也就牵扯出第二处小细节。2、如何确定第二个环？我们让重边数量最少，采用这样的策略：先找到树的直径，连接端点找到第一个环，在找第二个环之前，将直径路径上的边权全标记为-1，这样在第二次求树的直径时，就会尽可能避开这些边，也就是第一次找到的直径上的那些边，这样就可以使得重合边最少，最大节省路径。最后答案计算上，这样求得的 diameter2 就是第二个环上可以只走一次的边数，重合的边被第一个环认为只走一次，实际上走了两次，差一次没计算，也就是最终要补回来，即加上多走重边数，但是第二个环求解时，重边的边权设置为-1，正好可以抵消差的那一次，使得 diameter2 得到的是第二个环对答案的净贡献。也可以这样理解：第二条直径的边，如果出现在第一条直径上，那么对答案的贡献就是-1，因为重边需要多走一步；如果不在第一条直径上，那么对答案的贡献就是1，确实省下来的距离。计算时，边数的2倍 - d1 - d2 + 2 就是答案，加2是因为补了两条边。

​	在代码实现这里可以提前说一下：由于有负权边，第二次求直径需要用树型dp的方式；而第一次求解直径时，没有负权边（边权全是1），又需要路径信息，故采用两次 dfs 方式。建图时不需要输入边权，这样很难写，可以这样做：单独开一个布尔数组，出现在第一条直径上的点标记为 true，这样在第二次求解直径时，如果发现当前点和去往的下一个点时，这两个点均为 true，说明马上要走的这条边边权是-1，否则就是1。具体AC代码如下：

```c++
#include <bits/stdc++.h>
using namespace std;

#define MAXN 100005

int n, k;

int head[MAXN];
int to[MAXN << 1];
int nxt[MAXN << 1];
int cnt = 1;
void addedge(int u, int v) {
	nxt[cnt] = head[u];
	to[cnt] = v;
	head[u] = cnt++;
}

int s, e;
int d1, d2;
int dist[MAXN], last[MAXN];
bool diapath[MAXN];

void dfs(int u, int fa, int w) {
	last[u] = fa;
	dist[u] = dist[fa] + w;
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			dfs(to[e], u, 1);
		}
	}
}

void dp(int u, int fa) {
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			dp(to[e], u);
		}
	}
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			int w = diapath[u] && diapath[to[e]] ? -1 : 1;
			d2 = max(d2, dist[u] + dist[to[e]] + w);
			dist[u] = max(dist[u], dist[to[e]] + w);
		}
	}
}

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);
	
	cin >> n >> k;
	for(int i = 1, a, b; i <= n - 1; i++) {
		cin >> a >> b;
		addedge(a, b);
		addedge(b, a);
	}

	dfs(1, 0, 0);
	s = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[s]) {
			s = i;
		}
	}
	dfs(s, 0, 0);
	e = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[e]) {
			e = i;
		}
	}
	d1 = dist[e];

	for(int i = e; i != 0; i = last[i]) {
		diapath[i] = true;
	}
	memset(dist, 0, sizeof(dist));
	dp(1, 0);

	if(k == 1) {
		cout << 2 * (n - 1) - d1 + 1;
	}
	else {
		cout << 2 * (n - 1) - d1 - d2 + 2;
	}

	return 0;
}
```

### 选择<=s的路径使最远点到路径的距离尽可能小

​	题目链接：https://www.luogu.com.cn/problem/P2491

​	这题也是个难题，但是分析比较清楚简单。首先贪心的一个点：这样的路径一定在直径上。那么就在直径上选择路径，使得路径长度<=s。需要记录一个 maxdist 数组，前面也有这个数组的求法，该数组的作用是，求直径上的点到其子树的最大距离，除了直径上的子树。这样我们从直径的最后端开始一步一步向前找，这个过程维护路径长度<=s的一段区间，作为一个修建方案。对于任意一个修建方案，其他村庄到这条路径的最大值来自于：两个端点到这条路径的距离以及这个区间内的所有点最大的 maxdist 值，三者最大作为这个修建方案的答案。该区间边滑动，边取最小值，作为最后的答案。滑动窗口内的最大值用单调队列维护。详情见AC代码：

```c++
#include <bits/stdc++.h>
using namespace std;

#define MAXN 300005
typedef long long ll;

int n, s;
int dist[MAXN], last[MAXN];
int start, ed, diameter;
int pred[MAXN];
int maxdist[MAXN];
bool diaPath[MAXN];

int head[MAXN];
int to[MAXN << 1];
int nxt[MAXN << 1];
int weight[MAXN << 1];
int cnt = 1;
void addedge(int u, int v, int w) {
	nxt[cnt] = head[u];
	to[cnt] = v;
	weight[cnt] = w;
	head[u] = cnt++;
}

void dfs(int u, int fa, int w) {
	last[u] = fa;
	dist[u] = dist[fa] + w;
	pred[u] = w;
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(to[e] != fa) {
			dfs(to[e], u, weight[e]);
		}
	}
}

int maxDistExceptDia(int u, int fa, int c) {
	int ans = c;
	for(int e = head[u]; e != 0; e = nxt[e]) {
		if(!diaPath[to[e]] && to[e] != fa) {
			ans = max(ans, maxDistExceptDia(to[e], u, c + weight[e]));
		}
	}
	return ans;
}

int q[MAXN];
int h, t;
int suml, sumr;
int compute() {
	int ans = INT_MAX;
	for(int l = ed, r = ed; l != 0; l = last[l]) {
		while(r != 0 && (sumr - suml + pred[r] <= s)) {
			while(h < t && maxdist[q[t - 1]] <= maxdist[r]) t--;
			sumr += pred[r];
			q[t++] = r;
			r = last[r];
		}
		ans = min(ans, max(suml, max(diameter - sumr, maxdist[q[h]])));
		if(q[h] == l) h++;
		suml += pred[l];
	}
	return ans;
}

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);
	
	cin >> n >> s;
	for(int i = 1, u, v, w; i <= n - 1; i++) {
		cin >> u >> v >> w;
		addedge(u, v, w);
		addedge(v, u, w);
	}
	dfs(1, 0, 0);
	start = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[start]) {
			start = i;
		}
	}
	dfs(start, 0, 0);
	ed = 1;
	for(int i = 2; i <= n; i++) {
		if(dist[i] > dist[ed]) {
			ed = i;
		}
	}
	diameter = dist[ed];
	for(int i = ed; i != 0; i = last[i]) {
		diaPath[i] = true;
	}
	for(int i = ed; i != 0; i = last[i]) {
		maxdist[i] = maxDistExceptDia(i, 0, 0);
	}

	cout << compute();

	return 0;
}
```
