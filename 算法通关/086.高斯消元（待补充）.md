# 高斯消元

​	高斯消元法其实就是解二元或者多元方程组时消去未知数从而求出某个未知数的方法，本质是线性代数中系数矩阵的变换过程。

### 矛盾、多解、唯一解

​	上面的例子是三个未知数，三个方程，有唯一解。但是还有其他情况，例如解不唯一，或者无解。

​	什么时候解不唯一呢？就是化简到最后，1的个数与未知数的个数对不上，换句话说，主对角线上的1不全，同时解的部分也为0。在线性代数中，就是秩r < n（未知数个数的数量），此时解是不唯一的。那么什么时候无解呢？还是凑不齐1的情况，这时如果能找到任意一个主对角线的位置是0，对应的解的部分的值不是0，那么就无解，这个时候实际上是系数矩阵的秩小于增广矩阵的秩。

​	另外，我们刚才说，每来到新的列就忽略上一行，在下面的列寻找绝对值最大的系数，这个也有例外。那就是当前列对应全是0，此时找不到一个1换到当前行了，说明此时的未知数x是自由元，因为前面系数一定是0，取什么无所谓，此时再去下一列看的时候，就不能忽略当前行。这样能够区分多解和无解的情况。

​	为了说明上面的文字描述，我们来举个例子，如何区分矛盾与多解的情况：

![image-20260409131403740](typora-user-images\高斯消元\007.png)

​	对于这个方程组，来到第一行第一列，发现所有行都没有能换上来变为1的行，那就跳过，来到第二行第二列，此时我们不能忽略上面的行，也就是第一行的2应该被考虑到第二列寻找绝对值最大的系数这个过程中，如果不考虑，根据上面无解情况的叙述，主元位置是0，后面值不是0，那么就无解。但是事实上，如果考虑了这个2，将其换下来，会发现x2 = 3/2，x1后面的值是0，也就是x1为自由元，该方程组是多解，符合我们的预期的。

​	总结：在来到某一列要找绝对值最大的系数时，所有主对角线位置不是1的行都去参与pk。

### 高斯消元加法代码过程解析

​	首先，我们得到的矩阵一定是一个n * (n + 1)的矩阵，n个未知数，n个表达式。第n + 1列是各个方程的值。

​	大体流程是遍历所有的行，来到 i 行 i 列时，去找当前列中系数绝对值最大的行，将其与第 i 行交换，然后变为1。哪些行参与比较呢？主元位置不为1的行都参与比较。此时涉及到除法，存储时肯定用到double类型。

​	很容易观察到时间复杂度为O(n ^ 3)，n为扩充后的矩阵个数，为什么需要扩充？见下方表达式冗余和表达式不足部分。

```c++
const int MAXN = 105;
double mat[MAXN][MAXN]; // 存储的矩阵
double sml = 1e-7; // double类型会有误差，认为小于sml就是0

void gauss(int n) {
    for(int i = 1; i <= n; i++) {
        int maxv = i;
        for(int j = 1; j <= n; j++) {
            if(j < i && abs(mat[j][j]) >= sml) continue;
            if(abs(mat[j][i]) > abs(mat[maxv][i])) maxv = j;
        }
        swap(mat[i], mat[maxv]); // 交换第i行和当前列系数绝对值最大的一行
        if(abs(mat[i][i]) >= sml) {
            double tmp = mat[i][i];
            for(int j = i; j <= n + 1; j++) { // 先将当前行所有值除以tmp，使得当前列，即主元位置变为1
                mat[i][j] /= tmp;
            }
            for(int j = 1; j <= n; j++) { // 用第i行将当前列都消为0，每次来到第j行
                if(i != j) {
                    double rate = mat[j][i] / mat[i][i];
                    // 第j行所有元素都加上第i行的元素乘以某个倍数，该倍数使得第j行当前列为0
                    for(int k = i; k <= n + 1; k++) {
                        mat[j][k] -= mat[i][k] * rate;
                    }
                }
            }
        }
    }
}
```

### 表达式冗余和表达式不足

​	正常n个未知数n个方程，但是如果有n + 1个或者更多的方程，就是表达式冗余；如果有n - 1个或者更少的方程，就是表达式不足。这时也有唯一解、矛盾、多解的情况。

​	表达式冗余去补充未知数就可以了。比如三个未知数，给出了四个方程，我们并不知道哪个是冗余的，那就在每个方程后面都补充上 "+ 0 * x4" 即可；表达式不足，就补充表达式就可以了，我们认为，补充的表达式各个未知数前面的系数都是0，值也是0。然后可以继续套用上面的代码了。

### 主元与自由元

​	这里需要单独提到，多解情况最后的矩阵，有些未知数可以直接确定，而有些未知数的确定依赖于自由元。例如一个三行四列的矩阵，是最终版的，第一行的数据为：1，0，0，2，这时可以直接得出x1 = 2的结论，但是第二行如果是：0，1，1，3，第三行全0，这时不能直接确定x2与x3的值，需要确定自由元x3才能确定x2的值，所以主元可能被某些自由元影响到。

​	主元不会影响主元，因为住院处于不同行不同列。主元不会影响自由元，主元所在的列其余位置全是0。自由元不会影响自由元，因为选出自由元是因为当前列找不到一个不是0的行交换，即所有自由元系数均为0，不会相互影响到。

## 高斯消元解决异或方程组

***前置知识：解方程组、线性代数、异或运算、位图***

​	解法同加法方程组，区别的点在于，异或方程组的系数只有0和1，利用主元将同一列的其余元素都变为0也只需要异或操作，所以操作起来可能会更简单一些。代码在模板题中看。

​	模板题：https://acm.hdu.edu.cn/showproblem.php?pid=5833

​	这道题目说是模板题，但是还挺需要分析的，下面给他拆解为模板题。给出结论：如果一个数字是完全平方数，那么它可以拆成若干个质数相乘，其中每个质数出现都是偶数次。这样的道理也很简单，将每个质数分出一半在一个组，另一个组与它一模一样，所以是两个相同的数字相乘，这就跟题目给出的质数对应上了。那么接下来就看，选出的数字的每种质数因子的个数加起来是否为偶数，是的话就符合要求。遇到奇偶性判断的问题，应该很自然的想到异或运算。如何计算呢？对于arr中的每个数字，分解质因子，看其中每个质因子出现的次数是否为偶数，是的话就标记0，不是就标记1。最后对于每个数字，对应每个质数出现奇数还是偶数次，可以统计出一张二维表。我们让x1，x2，...，xn表示arr[1]，arr[2]，...，arr[n]上的数字要还是不要。先来分析第一种情况，举个例子，arr的第一个数字是12，它可以分解为2 * 2 * 3，所以12对于2这个因子的贡献是偶数，也就是12这个数字如果单看2这个质因子的话，要与不要均可，那么此时我们标记x1签的系数为0。继续分析，第二个数字假如是8，贡献了3个2，是奇数次，那么这时让x2前的系数为1。第三个数字假如是6，有一个2，也是有影响的，所以x3的系数也是1。第四个数字是60，x4的系数为0，第五个数字是21，x5的系数是0，把他们都异或起来，于是关于质因子2的一个方程就列出来了，我们让所有方程最后的结果统一为0，每个式子表示：对于每个质因子出现次数是偶数的判定，这样，解出x1到xn，就解出了每个数字要与不要了。
